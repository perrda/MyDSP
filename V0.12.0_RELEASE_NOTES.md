## ✨ What's New

### 1. IndexedDB Query Builder

SQL-like interface for IndexedDB operations:

```typescript
import { createQueryBuilder } from '@/utils/queryBuilder'

// Define schemas
const schemas = [{
  name: 'users',
  keyPath: 'id',
  autoIncrement: true,
  indexes: [
    { name: 'email', keyPath: 'email', unique: true },
    { name: 'age', keyPath: 'age' },
  ],
}]

// Create and initialize
const db = createQueryBuilder({ dbName: 'myapp', debug: true })
await db.init(schemas)

// Insert data
await db.from('users').insert({ name: 'John', email: 'john@example.com', age: 30 })
await db.from('users').insertMany([...])

// Query data
const users = await db
  .from('users')
  .where('age', '>', 25)
  .where('name', 'startsWith', 'J')
  .orderBy('age', 'desc')
  .limit(10)
  .get()

// Aggregation
const avgAge = await db.from('users').avg('age')
const maxAge = await db.from('users').max('age')
const grouped = await db.from('users').groupBy('city')

// Update & Delete
await db.from('users').where('age', '<', 18).delete()
await db.from('users').where('id', '=', 123).update({ verified: true })
```

**Features:**
- ✅ SQL-like query interface
- ✅ 10 comparison operators
- ✅ Index optimization
- ✅ Aggregation functions
- ✅ Pagination support
- ✅ Batch operations
- ✅ Migration system

### 2. Background Job Queue

Priority-based job processing with retry:

```typescript
import { createJobQueue, scheduleRecurring } from '@/utils/jobQueue'

// Create queue
const queue = createJobQueue({
  name: 'main',
  concurrency: 5,
  maxRetries: 3,
  debug: true,
})

// Register handlers
queue.process('send-email', async (data) => {
  await sendEmail(data)
  return { sent: true }
})

queue.process('generate-report', async (data, job) => {
  // Long-running task
  await generateReport(data)
  return { reportId: '123' }
})

// Add jobs
queue.add('send-email', {
  to: 'user@example.com',
  subject: 'Welcome',
  body: 'Hello!'
}, { priority: 'high' })

queue.add('generate-report', {
  userId: 'user-123'
}, { priority: 'normal', delay: 5000 })

// Listen to events
queue.on('job:completed', (job) => {
  console.log('Completed:', job.id, job.result)
})

queue.on('job:failed', (job) => {
  console.error('Failed:', job.id, job.error)
})

// Query & manage
const pending = queue.getJobs({ status: 'pending' })
const stats = queue.getStats()
queue.cancel(jobId)
queue.retry(jobId)

// Schedule recurring
const cancel = scheduleRecurring(
  queue,
  'cleanup',
  { maxAge: 7 * 24 * 60 * 60 * 1000 },
  24 * 60 * 60 * 1000 // Daily
)

// Shutdown
await queue.shutdown()
```

**Features:**
- ✅ Priority queues (4 levels)
- ✅ Concurrent processing
- ✅ Automatic retry + backoff
- ✅ Event system
- ✅ Persistent queue
- ✅ Job scheduling
- ✅ Statistics & reporting
- ✅ Pause/resume

### 3. Service Worker Manager

Advanced offline & push capabilities:

```typescript
import { createServiceWorkerManager } from '@/utils/serviceWorkerManager'

// Create manager
const swManager = createServiceWorkerManager({
  scope: '/',
  updateInterval: 3600000,
  enableBackgroundSync: true,
  enablePushNotifications: true,
  debug: true,
})

// Register
await swManager.register('/service-worker.js')

// Check for updates
await swManager.checkForUpdates()

// Skip waiting (activate new version)
await swManager.skipWaiting()

// Background sync
await swManager.registerSync('sync-data', { type: 'transactions' })

// Push notifications
const subscription = await swManager.subscribePush('VAPID_PUBLIC_KEY')

// Message passing
swManager.postMessage({ type: 'CACHE_URLS', urls: ['/api/data'] })

swManager.onMessage('SYNC_COMPLETE', (data) => {
  console.log('Sync completed:', data)
})

// Cache management
const cacheNames = await swManager.getCacheNames()
const size = await swManager.getCacheSize('api-cache')
await swManager.clearCache('old-cache')

// Status
console.log('State:', swManager.getState()) // 'active'
console.log('Controlling:', swManager.isControlling())
```

**Features:**
- ✅ Auto-registration & updates
- ✅ Background sync
- ✅ Push notifications
- ✅ Cache management
- ✅ Message passing
- ✅ Status monitoring
- ✅ Skip waiting support
